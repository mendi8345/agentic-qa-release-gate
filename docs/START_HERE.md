# Start here — 10-minute hands-on exercise

Your job is not to build the application. The baseline is already prepared.

## What you will do

1. Upload this repository to GitHub.
2. Confirm that the existing CI workflow is green.
3. Open a new issue using **Agent Task 01: Order status transitions**.
4. Assign the issue to an available coding agent.
5. Inspect its plan before focusing on the final code.

## Do not do yet

- Do not fix the transition bug yourself.
- Do not create a custom agent or skill yet.
- Do not expand the API.

## What to observe

- Which files the agent investigates.
- Whether its plan matches the acceptance criteria.
- Whether it distinguishes `400` from `404`.
- Whether its proposed tests cover the complete transition matrix.
